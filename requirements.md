# Discordログ抽出＆日報生成Bot 要件定義書

## 1. システム概要
指定したDiscordチャンネルの前日分メッセージを取得し、Claude APIで要約して日報（トピックスまとめ）として別チャンネルに自動投稿するDiscord Botシステム。

## 2. 目的・背景
- **情報共有の効率化**: チャンネルのタイムラインをすべて追いきれないメンバーへ向けて、1日1回前日の重要トピックスを要約して共有する。

## 3. 機能要件

### 3.1. Discordログ取得機能
- **対象チャンネル**: あらかじめ設定された特定のDiscordチャンネル（複数設定可能）。
- **取得タイミング**:
  - **定期実行**: 毎日決まった時間（デフォルト: 朝8時 JST）に実行。
  - **手動トリガー**: スラッシュコマンド `/create-summary` による任意実行。
- **取得対象データ**:
  - 投稿日時 (Timestamp)
  - 投稿者名 (Author)
  - メッセージ本文 (Content)
  - 添付ファイルのURL (Attachments)

### 3.2. 日報（トピックス）作成機能
- **処理対象**: 「実行日の前日（JST 0:00〜23:59）」に該当するメッセージ群。
- **要約処理**:
  - Anthropic API（Claude）を使用して要約。
  - 「主要なトピックス」「決定事項」「共有事項」「その他の動き」の4カテゴリで出力。
  - メッセージが大量の場合はチャンク分割して処理し、最終的に統合要約する。

### 3.3. Discordへの自動投稿機能
- **出力先**: 日報出力用に指定されたDiscordチャンネル（`SUMMARY_CHANNEL_ID`）。
- **投稿形式**: Markdown形式（太字・リスト・絵文字を活用）。
- **文字数制限対応**: 2,000文字を超える場合は段落単位で分割投稿。

## 4. 非機能要件

### 4.1. 実行環境
- **言語環境**: Python 3.10以上
- **動作環境**: 常時稼働環境（Render.com Worker、VPS 等）
- **主要ライブラリ**: `discord.py`, `anthropic`, `python-dotenv`

### 4.2. セキュリティ・シークレット管理
- Discord Bot Token、Anthropic API Key は `.env` で管理し、ソースコードに直接書かない。
- Botには必要最小限の権限（`Read Messages / View Channels`, `Read Message History`, `Send Messages`）のみを付与。

### 4.3. 運用上の留意点
- **LLMのコンテキスト上限**: メッセージが多い場合はチャンク分割処理で対応。
- **Discord投稿文字数制限**: 2,000文字超の場合は段落単位で複数投稿に分割。
- **Discord APIレートリミット**: メッセージ取得時に0.5秒の待機時間を挿入。

## 5. チャンネル設定（縁紡コミュニティ）

| 用途 | チャンネルID |
|---|---|
| 監視対象（ソース） | 要設定（`.env` の `SOURCE_CHANNEL_IDS`） |
| 日報投稿先 | 要設定（`.env` の `SUMMARY_CHANNEL_ID`） |

## 6. 処理フロー

```
定期実行 or /create-summary
    ↓
前日分メッセージを指定チャンネルから取得
    ↓
Claude APIで要約生成（大量時はチャンク分割→統合）
    ↓
SUMMARY_CHANNEL_ID に日報投稿
```
